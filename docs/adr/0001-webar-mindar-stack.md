# WebAR + MindAR을 'AR 리빌' 스택으로 채택 (네이티브·8th Wall·유료 매니지드 대신)

> **⚠️ 대체됨(superseded) → ADR 0002.** MindAR의 프레임별 추적이 모션 시 드리프트해, SLAM 추적을 위해 8th Wall OSS로 전환. WebAR 방향성은 유지.

NFC 태그가 "앱 설치 없이 아무 방문객이나 태그 → 즉시 카메라"를 열어야 하므로 네이티브 앱(ARKit) 대신 **WebAR**로 간다. 이미지추적 엔진은 무료·오픈소스이고 이미지추적에 특화돼 three.js 커스텀 글로우 셰이더 자유도가 높은 **MindAR**를 택한다.

## 맥락 / 트레이드오프

- **2026.2 Niantic이 8th Wall(수년간 최강 WebAR)을 SaaS 종료하고 엔진을 MIT 오픈소스로 공개**했으나, OSS 릴리스가 갓 나와 툴링·문서가 거칠고 이미지추적이 OSS 빌드에 포함되는지 미확인 → 프로토타입엔 MindAR이 빠르고 확실. *프로덕션 견고함이 필요해지면 8th Wall OSS를 재평가한다.*
- **유료 매니지드(Zappar/Blippar/Hololink)**는 뷰수 과금·구독비·셰이더 자유도 제한으로 제외.
- **네이티브 AR(ARKit)**은 추적이 더 견고하나 "이미 설치된 앱 보유자"만 가능 → 핵심 가치인 "태그하면 그냥 됨" 마법이 깨짐.

## iOS NFC 제약 (코드에 안 보이는 제약 — 설계에 반영)

- Safari는 **WebNFC 미지원**이나, 우리 방식(태그에 NDEF-URL 굽기 → iOS가 네이티브로 Safari 오픈)은 이와 무관하게 작동.
- **카메라가 켜져 있으면 백그라운드 NFC 읽기가 비활성** → "태그 먼저 → 그다음 카메라" 순서가 강제되고, AR 도중 재태그는 불가.
- URL을 **유니버설 링크**로 등록하지 않으면 태그 시 "Website NFC Tag" 배너를 1회 탭해야 함(프로토타입 무방, 프로덕션선 유니버설링크로 제거).

## 결과

- 카메라 접근 위해 **HTTPS(보안 컨텍스트) 필수**.
- 글로우 이펙트는 three.js(WebGL) 셰이더로 직접 구현 → 스타일 자유.
- 문양은 추적 제약(특징점 풍부·고대비·무광·비대칭)을 만족해야 함(플레이스홀더-퍼스트로 우회 후 디자인).
